## Information crawling

Always defer to the index file (`index.md`) for general/core information about this project. Before answering questions or exploring the codebase to reconstruct context, read `index.md` first and follow its links to the relevant category. The index is a hierarchy: categories → subtopics → leaf docs. Treat it as the single entry point. As new context docs are added, link them under the matching category in `index.md` rather than relying on filesystem search.
